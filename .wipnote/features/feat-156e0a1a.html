Description

Add lightweight observability for SQLite contention: count SQLITE_BUSY occurrences by command/subsystem, record effective journal mode, expose queue depth when the single writer is enabled, and add a stress command or test fixture that runs dashboard/indexer/hooks/CLI operations together. Use this as a release gate for Launch Readiness. Threshold tuning: the stress-fixture pass criterion is "zero SQLITE_BUSY from in-repo writer/indexer/hook paths." That is distinct from "zero SQLITE_BUSY anywhere" — external producers (MCP, user-installed tools) can still produce BUSY and are not gated by this slice (see slice 5 scope clarification).
