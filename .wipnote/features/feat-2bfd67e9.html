col.Complete(id) flushes the canonical HTML to disk at cmd/wipnote/workitem.go:258, BEFORE commitWipnoteArtifact at :329, so an in-process status-field revert is physically impossible. New design: capture pre-commit HEAD before col.Complete; add commitWipnoteArtifactStrict in cmd/wipnote/workitem_commit.go (a variant that RETURNS an error) and call it ONLY from the complete path. After it runs, assert two invariants: (a) the item's own artifact path is clean (no unstaged/uncommitted .wipnote//.html), and (b) git log -1 --format=%H -- advanced vs the captured pre-commit HEAD. On failure: perform a COMPENSATING RE-OPEN transition — a second state write, col.Edit(id).SetStatus("in-progress").Save() (or the equivalent revert transition) — so the item ends in-progress, not done, then exit non-zero printing the exact remediation command. This second write has its own side-effect surface (lineage trace edges, session events, agent attribution); the design accepts that surface and done_when asserts the item ends in-progress with coherent side effects. The existing non-fatal commitWipnoteArtifact (workitem_commit.go:62-65, called non-fatally at :322-323) is UNCHANGED for all other callers. Leave checkUncommittedSourceCompleteGate (the outside-.wipnote dirty-source guard) untouched in behavior.

# Spec: Transactional wipnote * complete (compensating re-open on commit failure)

## Problem
Highest-leverage, evidence-backed. The existing guard only rejects
unrelated dirty source OUTSIDE .wipnote/; it never verifies the item's
OWN artifact commit landed. Confirmed live this session - Codex left 6
items done yet evaporable; bug-6a925774's completion only persisted
because it happened to go through the proper path.

## Decisions
**Trigger:** complete path persists canonical HTML via col.Complete at
workitem.go:258 then calls the commit at :329; commit can silently fail
leaving an evaporable done item.
**State:** durable git state - pre-commit HEAD captured before
col.Complete plus git log -1 --format=%H -- <artifact>; no session
state (per the file/branch-state hook rule).
**Payload:** on failure a compensating re-open transition
(SetStatus in-progress) plus non-zero exit + exact remediation command;
commitWipnoteArtifactStrict is used by the complete path only.
**Rejected:** in-process status-field revert (rejected — physically
impossible: col.Complete flushes the canonical HTML at workitem.go:258
BEFORE commitWipnoteArtifact at :329, so there is no in-memory field to
revert); breaking the non-fatal commitWipnoteArtifact contract for all
callers (rejected — workitem_commit.go:62-65 contractually returns nil
on failure for YOLO/worktree callers; add a strict variant instead);
more CLAUDE.md/AGENTS.md prose telling agents to verify commits
(rejected - advisory text is exactly the failure mode; CLAUDE.md
philosophy says fixes land in the binary/hooks, never descriptive docs).
