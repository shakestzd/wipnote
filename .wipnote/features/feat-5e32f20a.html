Description

This is the slice where `htmlgraph serve` transitions from 'single-project server' to 'parent with reverse proxy and projects-landing'. Multi-part change: **(1) Drop the `--global` flag entirely.** Delete the flag definition from the serve cobra command. `htmlgraph serve --global` becomes an unknown-flag error (cobra default). Users who typed `--global` get a clear cobra error and can re-run without the flag. **(2) Merge `runGlobalServer` body into `runServer`.** `runServer` now always starts the parent process: builds the global mux (landing SPA + `/api/mode` + `/api/projects` + `/p//*` proxy), instantiates the `childproc.Supervisor` from slice 2, starts the idle reaper goroutine, and calls `http.ListenAndServe(addr, mux)` on the configured `--port` (default 8087). The old single-project behavior is gone from `runServer` — it moved to the hidden `_serve-child` command in slice 1. Delete the now-dead `runGlobalServer` function. **(3) Proxy handler registration.** Register `mux.Handle("/p/", corsMiddleware(http.HandlerFunc(proxyHandler)))` where `proxyHandler` parses `/p//`, validates the project ID (reject empty, `.`, `..`, `/`, null bytes, non-alphanumeric+underscore+hyphen — returns 400 on validation failure), looks up the project in the registry to get `ProjectDir` (returns 404 if unknown), calls `supervisor.GetOrSpawn(r.Context(), projectID, projectDir)` (returns 502 on spawn failure), strips the `/p/` prefix from `r.URL.Path` via `path.Clean("/" + rest)` not string manipulation, calls `child.LastRequest.Store(time.Now().Unix())`, and forwards via the pre-built `child.proxy` (already has `FlushInterval = 100 * time.Millisecond` for SSE from slice 2). The proxy's `ErrorHandler` (set at construction time in slice 2) returns 502 on child unreachable. **(4) Current-project auto-detection (Option B).** Before starting the HTTP server, call `findHtmlgraphDir()` to check if CWD contains a `.htmlgraph/` directory. If yes, derive the project ID via `SHA256(absPath)[:8]`, auto-register it in the registry if not already present (same `registry.Upsert` call the `PersistentPreRunE` uses), and extend the startup banner to print BOTH URLs: `http://localhost:/` (doorway) and `http://localhost:/p//` (current project deep link). If CWD has no `.htmlgraph/` or the registration fails, silently print only the doorway URL. The detection code is ~10 lines at the top of `runServer` before the mux is built. Do NOT auto-open the browser — just print the URL for the user to click. **(5) Graceful shutdown.** Plumb `supervisor.Shutdown(ctx)` through `http.Server.RegisterOnShutdown` so when the parent receives SIGINT/SIGTERM, it cleanly kills all children before exiting.
