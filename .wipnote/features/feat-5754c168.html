Description

Add 'htmlgraph cleanup ghost-sessions [--project-dir ] [--dry-run]' CLI subcommand that safely removes session rows which have NO canonical HTML file AND no messages, tool_calls, or agent_events. Must respect the HTML-canonical invariant: if a session has an HTML file on disk, it stays regardless of whether the DB shows zero events. Context: during v0.54.0 verification, I ran a raw SQL DELETE to clean up 'ghost' session rows in shakestzd and accidentally deleted 7 sessions that had HTML files but no events. The HTML-canonical reindex recovered them, but the incident surfaced a gap: there's no CLI-level safe cleanup path that joins filesystem state with SQLite state. This feature fills that gap. Logic: 1. Walk .htmlgraph/sessions/*.html to build the set of session_ids that have canonical HTML files 2. Query the sessions table for all session_ids 3. Identify true ghosts: rows that have NO HTML file AND zero rows in messages + tool_calls + agent_events 4. In --dry-run mode: print the list with per-row reasons (no HTML, no events) 5. Without --dry-run: DELETE the rows with progress output, report count 6. Never delete a session that has an HTML file — HTML is canonical Files to create: cmd/htmlgraph/cleanup.go, cmd/htmlgraph/cleanup_test.go Files to modify: cmd/htmlgraph/main.go (register cleanupCmd()) Estimated effort: 100-150 lines Go + tests.
