Description

Add a Codex agent translator that converts canonical `AgentAsset` values into `.toml` files matching the documented Codex custom agent schema: `name`, `description`, and `developer_instructions` are required, with optional `model`, `model_reasoning_effort`, `sandbox_mode`, `nickname_candidates`, and `skills.config` only when there is a clear Codex mapping. The initial mapping should: - Emit `developer_instructions` from the markdown body plus any `initialPrompt` text. - Map Claude model aliases conservatively: `haiku` to an inexpensive Codex model only if the repo already has a Codex model policy; otherwise omit `model` and inherit from parent. - Map Claude `tools: [Read, Grep, Glob]`-style allowlists onto Codex's `sandbox_mode` field — reader-class agents (e.g. `reader.md`, `researcher.md`) become `sandbox_mode = "read-only"`; writer-class agents (`haiku-coder.md`, `sonnet-coder.md`, `opus-coder.md`, `test-runner.md`) become `sandbox_mode = "workspace-write"`. - Omit Claude-only fields such as `color`, `disallowedTools`, and `maxTurns` — Codex has no documented equivalents. - Prefix or namespace names only if needed to avoid collisions; prefer stable names like `wipnote-sonnet-coder` for user-level agents.
