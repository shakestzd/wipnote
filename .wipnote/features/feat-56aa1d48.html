Description

New CLI subcommand htmlgraph otel-collect --session-id --project-dir --listen :0. On start, constructs a receiver.Receiver wired to NDJSONSink rooted at .htmlgraph/sessions//, chooses an ephemeral port via net.Listen(:0), and prints the chosen address on stdout using the existing serve_child.go:70 handshake convention: htmlgraph-otel-ready port= as a single line (no prefix/suffix writes). All internal logging routes to stderr so the stdout handshake line is never mingled with diagnostics. Handles SIGTERM gracefully: flush pending batch (bounded 3s or less), close sink, exit 0. Implements the Q3 layered-all-three safety net third layer: if HTMLGRAPH_OTEL_IDLE_TIMEOUT (default 5 minutes) elapses with no incoming OTLP traffic, the collector self-exits 0 after flushing. Emits a collector_start NDJSON line on boot. Registered as a cobra subcommand in cmd/htmlgraph/main.go:248-273. ## Why The collector must run as a separate process to achieve lifecycle independence from serve. Reusing the existing htmlgraph-*-ready port= handshake from serve_child.go:70 means launcher port-capture code (slice 3) follows an already-proven pattern rather than inventing a new one. SIGTERM drain is the only shutdown path that preserves in-flight spans. Forcing the launcher to pass session-id via flag closes the HTMLGRAPH_SESSION_ID env-leak misattribution at runner.go:187. The idle-timeout is the third layer of Q3 defense-in-depth policy and lives in the collector itself rather than in any hook, because only the collector can observe its own OTLP traffic silence. Supports goal Launcher-owned session identity. ## Done When - htmlgraph otel-collect --session-id 01HK... --project-dir /tmp/proj --listen :0 prints a single htmlgraph-otel-ready port= line on stdout within 500ms of start; all logging routes to stderr. - POSTing an OTLP protobuf to the printed port produces a corresponding NDJSON line in .htmlgraph/sessions/01HK.../events.ndjson within 500ms. - SIGTERM drains the pending batch and exits 0 within 3s; SIGKILL accepted without corrupting NDJSON beyond the last partial line. - HTMLGRAPH_OTEL_IDLE_TIMEOUT (default 5m) with no OTLP traffic triggers graceful self-exit 0 after flushing; overridable via env var for tests. - collector_start event is the first line of the NDJSON file and contains htmlgraph_sid, pid, port, ts.
