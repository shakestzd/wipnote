Description

Pilot evaluation infrastructure — produces the evidence needed to make a GREEN/YELLOW/RED promotion decision for a follow-on hook-driven plan. 1. **Empirical cost measurement (one-time pre-merge task).** Run `htmlgraph compliance auto ` on one real completed feature with a non-trivial spec. Record the measured `total_cost_usd` from the JSON output. If materially different from the $0.03/call estimate, update `q-model-default` recommendation and re-baseline cost framing in `q-pilot-success-bar`. 2. **Retrospective batch baseline.** Run `htmlgraph compliance auto --batch-since 2026-01-01` (slice 1's batch mode) against historical features in this project. Output of that run becomes the day-0 evaluation dataset. Capture the run's aggregate cost. 3. **`htmlgraph compliance history ` command.** Reads prior compliance runs from the feature's findings section's ` ` blocks (last 5 retained by slice 1's idempotent replace, when slice 1's design is extended in a future iteration to preserve history) AND from the section's `data-*` attributes for the most recent run. Prints a table: timestamp, model, score, cost, spec-hash, diff-truncated. Supports `--json` for evaluation script consumption. 4. **Review checkpoint runbook** — `docs/runbook/compliance-pilot-evaluation.md` describing: - **Day 0 baseline:** measured cost + retrospective batch dataset - **Day 7 checkpoint:** count of `compliance auto` invocations across dogfood users, sample of 5 random findings reviewed for plausibility, decision to continue or adjust prompt - **Day 14 checkpoint:** full quality audit — manual review of 10 findings against engineer judgment, score-agreement target ≥70% - **Decision criteria:** GREEN (all gates pass) → spawn hook-driven follow-on plan; YELLOW (signal noisy) → tune prompt, extend pilot 7 days; RED (consistent disagreement or unmanageable cost) → kill pilot, document why 5. **Dogfood activation.** Project-level shell-alias or runbook entry directing dogfood users to run `htmlgraph compliance auto ` after each `feature complete`. Since the hook is out of scope, "activation" means establishing muscle memory and reporting cadence. The runbook explicitly states the 14-day window and the success criteria.
