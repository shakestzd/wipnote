Description

End-to-end integration test that proves HTML is truly canonical for all session types after slices 1-3 land. The test verifies: **(a) Ingest produces reindex-compatible HTML.** Ingest a fixture JSONL transcript via `htmlgraph ingest`, verify SQLite row exists, verify HTML file exists, nuke the SQLite DB, run `htmlgraph reindex`, verify the session row is rebuilt with identical agent_events count and metadata. **(b) Live hook HTML round-trips.** Simulate a SessionStart → PreToolUse → PostToolUse → SessionEnd sequence via the hook CLI commands, verify the resulting HTML file, nuke SQLite, reindex, verify the rebuilt row matches. **(c) Orphaned events are recovered.** Simulate a PreToolUse without a matching PostToolUse (direct insert into agent_events with `status='started'`), advance time past the threshold, run `htmlgraph sweep orphaned-events`, verify the HTML file has a new synthetic ` ` entry, nuke SQLite, reindex, verify the rebuilt row shows `status='aborted'`. **(d) Migration handles orphans.** Create a tmpdir project with SQLite rows and NO HTML files (simulating legacy state), run `htmlgraph migrate sessions`, verify HTML files are created, nuke SQLite, reindex, verify all sessions rebuild. **(e) Concurrent writers.** Spawn N goroutines simulating concurrent hook subprocesses calling `appendEventToSessionHTML` on the same session file. Assert the resulting file is (1) well-formed and parseable by goquery AND (2) contains ALL N events — not just 'no crash' or a subset. This is the load-bearing acceptance criterion for R1 (concurrent flock writers landing all events). Spawn 20+ goroutines across three writer types — ingest render, live PostToolUse append, and sweep append — all targeting the same session HTML file. Verify every single event's `data-event-id` is present in the final file. This verifies the existing flock path still works after any refactoring in slices 1-3. All tests under `//go:build integration` build tag so they don't slow the default `go test ./...` run. CI runs them in a separate job.
