Description

Audit every writable SQLite call site and classify it as one of: daemon-routed, canonical-only, reindex-only, migration-only, or intentional CLI work-item mutation. Add an enforcement test/static check that fails when hook, collector, indexer, or event-capture paths open a writable DB directly outside the approved boundary. Document the approved write paths in code comments near the DB open helpers. Scope clarification: the boundary test scans first-party Go source under `cmd/`, `internal/`, and `plugin/`. MCP servers, third-party plugins, and external tools that open the DB file directly are explicitly OUT of scope — those are user-installed and uncontrollable from the wipnote repo. Document this as a known limitation; if observed in practice, file a follow-up bug rather than expanding this slice.
