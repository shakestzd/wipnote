Description

Add a migration subcommand that finds every session row with no corresponding HTML file and renders one using slice 1's `renderIngestedSessionHTML` helper. The migration runs once and is not invoked automatically on every CLI call. **(a) New CLI subcommand.** `htmlgraph migrate sessions [--project-dir ] [--dry-run]`. Scans `.htmlgraph/sessions/*.html` to build the set of session IDs that already have HTML, then queries the `sessions` table for rows NOT in that set. For each orphan, loads its messages and tool_calls from SQLite and constructs a `ParseResult`-like struct that can be passed to `renderIngestedSessionHTML`. **(b) Data sources.** The rendered HTML is built from what SQLite has. If the original JSONL transcript is still available via `~/.claude/projects/`, prefer parsing it fresh via the ingest path (higher fidelity). Otherwise, fall back to rendering from the SQLite rows (lower fidelity — missing some event context that was never stored). Report which sessions used which source in the output. **(c) Dry-run mode.** `--dry-run` prints the list of orphan sessions and their data source without writing any files. Lets the user inspect before committing. **(d) Progress output.** For each orphan, print `session-id: source= events=N -> .htmlgraph/sessions/.html`. Summary at the end: `Migrated N sessions (M from JSONL, K from SQLite fallback, L errors)`. **(e) Error handling.** If a single session fails to render (e.g., malformed SQLite row, missing messages), log the error and continue with the next. Do not abort the whole migration. Final report shows errors at the end. **(f) Idempotency.** Running the migration twice is a no-op on the second run — the skip-if-exists check from slice 1 means already-migrated sessions are left alone. Useful for users who want to safely re-run. This closes Gap 3.
