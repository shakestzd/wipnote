# Spec: Gate-record-required completion (session-local indexable gate record)

## Problem
Kills -run-filtered partial validation presented as a passing gate.

## Decisions
**Trigger:** complete invoked on a code-bearing item with no valid
session-local passing gate record at complete time.
**State:** derived read-index row (gate_records table) scoped to the
current session/worktree, re-validated immediately before complete; NOT
cross-session SHA-pinned and NOT canonical .wipnote HTML.
**Payload:** wipnote check --gate runs the real full build/vet/test
(manifest-detected) and writes the indexable record; allowlist hits
reported, never silently absorbed; slice-7 JOINs it for the ran-gate
dimension.
**Rejected:** cross-session HEAD-SHA-pinned record (rejected — slice-5
reconcile auto-commit mutates HEAD and would strand a stale-but-passing
record; session-local + re-check-at-complete avoids the interaction);
storing the record as canonical .wipnote HTML (rejected — violates the
HTML-only canonicity constraint; a derived read-index row is correct and
feeds slice-7); a blanket env-flake skip (rejected — masks genuine
failures; each allowlist entry needs a recorded justification);
per-harness gate scripts (rejected — drifts, defeats parity).
