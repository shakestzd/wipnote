Currently each turn in the activity feed renders TWO child subtrees: hook-derived rows (from agent_events via renderEvent) and OTel-derived rows (from otel_signals via renderSpan, visually distinct with blue border). Now that OTel gives correct hierarchy (subagent tool calls nest natively, no tracer.go custom logic needed), we can collapse to a single tree per turn. Fall back to hook rendering only when no OTel data exists (pre-OTel sessions).
