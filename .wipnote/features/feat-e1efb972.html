Description

Fix the visible flash where subagent tool calls render as turn-root orphans until the parent Agent/subagent_invocation span arrives ~minutes later. Root cause: OTel emits spans on END; Agent span has long duration and arrives last. Fix design: (1) SubagentStart hook writes a pending_subagent_starts row keyed on agent_id with agent_type, description, cwd, session_id, parent_trace_id. (2) Extend writeSubagentEnvVars in internal/hooks/subagent_start.go to also inject OTEL_RESOURCE_ATTRIBUTES=htmlgraph.agent_id= so every subagent-emitted OTel span carries htmlgraph.agent_id as a resource attribute. (3) In the receiver writer (internal/otel/receiver/writer.go), when a span arrives whose parent_span is not yet in otel_signals, read htmlgraph.agent_id from resource attrs, look up pending_subagent_starts, UPSERT a placeholder otel_signals row keyed by the unknown parent_span with canonical=subagent_invocation, tool_name=Agent, _pending=1, attributes pre-filled from the pending_subagent_starts row. (4) When the real Agent span arrives later (same span_id), the existing UPSERT path fills duration/status/tokens/cost in place and clears _pending. Parallel subagents disambiguate cleanly via agent_id (unique per dispatch). Dashboard sees parent row present from first paint; no flash. Supersedes bug-73eb8412.
