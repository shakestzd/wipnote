Add wipnote reconcile --strict plus a Stop/SessionEnd hook handler (internal/hooks/session_end.go and the Stop path at internal/hooks/missing_events.go:101) that runs it on session exit. reconcile detects three classes: done-but-uncommitted items (auto-commit the artifact — deterministic bookkeeping — and report), generator-touched-without-build-ports drift (reuse slice-2 check-ports), and started-but-orphaned items. The Stop handler has an EXPLICIT never-block contract (missing_events.go:114 and :125) and BlockExit2Error is only wired into TaskCompleted (missing_events.go:305, task_completion_gate.go:15); Gemini SessionEnd routes through session-end and Codex TaskComplete maps to stop, both sharing the non-blocking path. New design: add an explicit HARNESS DISCRIMINATOR in the reconcile/Stop path — return BlockExit2Error (exit-2 block) ONLY when harness=="claude" (deliberately amending the missing_events.go:114/125 no-block contract); for Gemini and Codex emit a DURABLE structured warning (persist to a warnings log under .wipnote/ or the read index) AND surface it at next SessionStart (internal/hooks/session_start.go), NOT context-injection-only, so the user-never-returns case is still recorded. reconcile auto-committing a done-but-uncommitted artifact must NOT strand a stale-passing slice-4 gate record — slice-4 records are session-local and re-checked at complete, so the interaction is closed.

# Spec: Session-exit reconciliation (reconcile --strict + harness-discriminated Stop/SessionEnd)

## Problem
Defense-in-depth so a session that violated invariants is caught before
the agent walks away, on every harness.

## Decisions
**Trigger:** session Stop/SessionEnd with done-but-uncommitted items,
generator-without-build-ports drift, or orphaned started items.
**State:** durable - git working tree, artifact commit state, slice-2
port diff, plus a durable warnings record for the weak-harness path;
not ephemeral session history.
**Payload:** auto-commit deterministic bookkeeping + report; harness
discriminator returns BlockExit2Error only when harness==claude;
durable warning persisted + surfaced at next SessionStart on
Codex/Gemini.
**Rejected:** relying on the Stop handler to block (rejected — it
explicitly never blocks at missing_events.go:114/125 and
BlockExit2Error is only wired into TaskCompleted; an explicit harness
discriminator is required); context-injection-only warning for
Codex/Gemini (rejected — loses the user-never-returns case; persist
durably and surface at next SessionStart); cross-session SHA-pinned
gate records (rejected — reconcile auto-commit mutates HEAD; slice-4 is
session-local to close the interaction); per-harness bespoke reconcile
(rejected — drifts, defeats parity; one shared entrypoint invoked by
the per-harness hook matrix).
