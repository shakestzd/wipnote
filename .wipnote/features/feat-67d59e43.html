Description

Update `internal/storage.CanonicalDBPath` and DB diagnostics so wipnote selects a SQLite cache root backed by a WAL-safe filesystem when possible. Prefer explicit `WIPNOTE_DB_PATH`; otherwise probe candidate roots such as `XDG_RUNTIME_DIR`, `/tmp`, and `os.UserCacheDir()`. Record why a path was chosen and expose journal mode/fstype in `wipnote status` or a diagnostic helper. If no safe path exists, intentionally choose DELETE mode and rely on the single-writer design rather than pretending concurrency is solved.
