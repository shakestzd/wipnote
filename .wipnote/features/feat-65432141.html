Description

In cmd/htmlgraph/otel_collect.go runOtelCollect, spawn internal/otel/indexer.New(...).Start(ctx) as a goroutine after the HTTP server starts and before awaitShutdown blocks. Coordinate goroutine lifecycle via a `sync.WaitGroup` plus an error channel (stdlib only — `golang.org/x/sync/errgroup` is not in `go.mod` requires) so server + indexer share lifecycle and a cancellation propagates to both. The indexer's *sql.DB injection is decided in q-db-handle-injection (recommended: collector opens via db.Open and injects via WithDB). Pass the indexer the `.htmlgraph/` subdirectory path: `htmlgraphDir := filepath.Join(projectDir, ".htmlgraph")` — `indexer.New` does NOT accept the project root. DB handle injection follows q-db-handle-injection (recommended option-b): collector calls `db.Open()` and passes the handle to `indexer.New(...).WithDB(db)`. The collector also adds `internal/db` to its imports for the first time. ## Why The indexer code in internal/otel/indexer exists and is unmounted; this is the missing wiring that makes spans visible in SQLite. Without it, every /api/otel/* endpoint and the dashboard return empty. ## Done When - runOtelCollect launches an indexer goroutine when the collector starts - SIGTERM cancels the indexer ctx and the goroutine returns from Start within the shutdown window (covered fully by slice-2) - An OTLP span POSTed to the collector appears in SQLite otel_signals within one indexer tick (~500ms)
