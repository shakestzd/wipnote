Description

New test file `cmd/htmlgraph/serve_global_proxy_test.go` (under `//go:build integration`) with three integration tests and a shared `TestMain`. **`TestMain`** builds the htmlgraph binary once at test startup via `exec.Command("go", "build", "-o", filepath.Join(tempDir, "htmlgraph"), "./cmd/htmlgraph")` and stores the path in a package-level variable. All three tests reuse this binary — no second go build. This is REQUIRED (not optional per feasibility critic A10/C8) to keep CI under 60s; a per-test go build would cost 5-15s each. The same `TestMain` pattern applies to `internal/childproc/childproc_integration_test.go`. **Test A (cross-project contamination proof):** create tmpdir projectA and projectB each with initialized `.htmlgraph/`; register both in a temp `~/.local/share/htmlgraph/projects.json`; start the server with `supervisor.binPath` pointing at the test-built binary; via the proxy, POST `/p//api/workitem/create` to create feature X in projectA; open projectB's SQLite file DIRECTLY from disk (not via any proxy) via `dbpkg.OpenReadOnly` and assert no `work_items` row for feature X; open projectA's file and assert feature X IS there. **Test B (crash recovery):** spawn a child via the supervisor; kill the child with `cmd.Process.Signal(syscall.SIGKILL)` from the test; send the next request through the proxy; assert a fresh child is spawned (new PID) and the request succeeds. **Test C (idle reap):** create a supervisor with `idleTimeout = 200ms`; spawn a child; wait 300ms without sending any requests; assert the reaper killed the child and the map entry is gone. Default `go test ./...` (without `-tags=integration`) does NOT run these — CI adds a separate job with `-tags=integration`.
