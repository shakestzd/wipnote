Description

Extend the `cmd/wipnote/codex.go` setup path so `wipnote codex --init`, `wipnote codex --dev`, and normal `wipnote codex` ensure generated Codex custom-agent TOML files are present in the user's Codex agent directory. Distinguish staging from install: - **Staging**: the generator writes TOML to a build-output path under the wipnote source tree (e.g. `dist/codex-agents/.toml`) for inspection and CI. This is NOT where Codex loads agents from. - **Install**: at launch time, copy generated TOML into `~/.codex/agents/` (user-global) — Codex's documented agent location. `os.MkdirAll` the directory if absent (it is not present by default). Use a stable filename per agent (e.g. `wipnote-sonnet-coder.toml`). In dev mode, force-refresh on each launch. In normal mode, install idempotently: read the existing file's ownership marker; only overwrite if the marker matches wipnote's (so user-authored files with the same name are never clobbered). Add a clear ownership marker as a TOML comment block at the top of every generated file: # wipnote-managed: do not edit by hand. version=1 # source: # generated: wipnote treats absence of this marker as user-authored and refuses to overwrite, even in dev mode.
