Add to HarnessConfig in registry.go: ```go type OtelEnvFunc func(port int, sessionID string) []string // OtelEnv returns the env vars to inject before launching this harness. // nil is valid only for Claude (Claude Code sets its own OTel config). OtelEnv OtelEnvFunc ``` Populate OtelEnv in registry_codex.go and registry_gemini.go: - Codex: returns ["OTEL_EXPORTER_OTLP_ENDPOINT=...", "OTEL_SERVICE_NAME=codex-cli", ...] The env var name strings are const-declared within registry_codex.go. - Gemini: returns ["GEMINI_TELEMETRY_ENABLED=true", "GEMINI_TELEMETRY_OTLP_ENDPOINT=...", ...] Same pattern in registry_gemini.go. - Claude: OtelEnv left nil (Claude Code injects its own OTel config). Add BuildAgentEnv() method to HarnessConfig: ```go func (c *HarnessConfig) BuildAgentEnv() []string { return []string{ "WIPNOTE_AGENT_ID=" + c.AgentID, "WIPNOTE_AGENT_TYPE=" + c.AgentID, } } ``` Both launcher files call this instead of hardcoding the env var assembly. Document that OtelEnv must be non-nil for Codex and Gemini registrations — add a panic in GetByHooksHarness if a non-Claude harness has nil OtelEnv and the caller requests OTel env vars.
