Description

New package `internal/childproc/` with `childproc.go` + `childproc_test.go`. Define `type Child struct { ID string; ProjectDir string; Port int; PID int; cmd *exec.Cmd; LastRequest atomic.Int64; cancel context.CancelFunc; exitC chan struct{}; proxy *httputil.ReverseProxy }`. Define `type Supervisor struct { mu sync.Mutex; children map[string]*Child; spawnGroups map[string]*sync.Once; binPath string; idleTimeout time.Duration; spawnTimeout time.Duration }` with per-project `sync.Once` to handle concurrent cold-spawn requests without herd contention on the main mutex. Implement `NewSupervisor(opts Options) *Supervisor` with `binPath = os.Executable()` by default. Implement `(*Supervisor) GetOrSpawn(ctx context.Context, projectID, projectDir string) (*Child, error)` which spawns via `exec.CommandContext(cancelCtx, s.binPath, "--project-dir", projectDir, "_serve-child", "--port", "0")`. **The arg order is load-bearing**: `--project-dir` is a persistent flag on the root cobra command (main.go:37) and must appear BEFORE the `serve` subcommand, not after, or cobra rejects it (feasibility critic A8). Capture stdout via `cmd.StdoutPipe()`, call `cmd.Start()`, launch a goroutine that scans stdout with `bufio.Scanner` for exactly one line matching `htmlgraph-serve-ready port= pid=` with a 5s deadline. After the handshake, the scanner goroutine drains the pipe into `io.Discard` forever so the child doesn't block — though the child should have already redirected its own stdout to a log file per slice 1's requirement. Populate `Child.Port` and `Child.proxy` (pre-built `httputil.NewSingleHostReverseProxy` pointed at `http://127.0.0.1:` with `FlushInterval = 100 * time.Millisecond` for SSE), and return. Second goroutine per child calls `cmd.Wait()`, closes `exitC`, removes from the map under the mutex. Implement `Shutdown(ctx)` which SIGTERMs all children in parallel, waits with deadline, SIGKILLs survivors. Implement `RunIdleReaper(ctx)` ticking every 60s. The closest prior-art pattern is `internal/planchat/backend.go:174` (exec.CommandContext + StdoutPipe + Start + scan + Wait) — model the scanner goroutine after it. Use `os.Executable()` not `os.Args[0]` (plan_cmds.go:268 uses os.Args[0] which is wrong for moved binaries / symlinks).
