SessionStart and SessionResume hooks currently emit ~15 lines of context on every fire. Much of it duplicates the system prompt (CLI reference block, orchestrator-directives pitch) which the launcher already injects via CLAUDE.md. Scope (pass 1, small/safe): 1. Drop the CLI reference block (htmlgraph CLI — feature|bug|spike|… line) from SessionStart output. Already surfaced via /htmlgraph:orchestrator-directives-skill and CLAUDE.md system prompt. 2. Drop the orchestrator-directives pitch line. Same reason. 3. Limit the open-items list from all-in-progress to the 3 most-recently-touched items (sort by updated_at DESC, limit 3). Add tail line 'N more — htmlgraph wip' when truncated. 4. Add serve health check: TCP-probe configured OTLP port (default 4318). When HTMLGRAPH_OTEL_ENABLED=1 and the port isn't bound, spawn htmlgraph serve detached (same logic as the launcher autostart added in claude_serve_autostart.go). Emit one status line: 'otel receiver: http://127.0.0.1:4318' or 'otel receiver: starting' / 'otel receiver: down'. Net effect: cut ~15 lines of output to ~5. No behaviour regressions — every useful piece of info stays, just once each.
