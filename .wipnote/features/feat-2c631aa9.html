Description

Add filesystem hygiene so the OTel indexer (and any other consumer that enumerates `.wipnote/sessions//`) never processes session directories that have no corresponding row in the `sessions` table. Concretely: 1. In `internal/otel/indexer/indexer.go`, gate `discoverSessions()` on a DB check: skip directories where `sessions.session_id` is absent. 2. Provide `wipnote cleanup orphan-sessions` that lists candidate orphan directories (>= 24h old, no DB row, no recent writes) and prompts before deletion. Default behavior is list-only. 3. Decide a retention policy in code (e.g. 14 days) after which orphan session NDJSONs are auto-deleted by a scheduled cleanup task in the writer service (slice 6 owns scheduling). Today the indexer eagerly tails every `events.ndjson` it finds. Two sessions in production were orphan (no DB row) and the indexer kept retrying them every 30s, contributing to the BUSY loop. Even with the 4MiB budget (commit e16095482) and the future single-writer queue, spending writer cycles on orphan data is waste.
