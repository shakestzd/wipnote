Problem

Anthropic shipped --permission-mode auto in Claude Code v2.1.111 (2026-04-16): a classifier model pre-reviews each tool call and blocks destructive patterns (curl|bash, force-push, mass deletion, IAM grants, etc.). HtmlGraph's PreToolUse guards are orthogonal — domain-specific work-discipline controls automode has no concept of. There are gaps on both sides: 1. **No first-class `htmlgraph auto` subcommand.** The `launchClaudeAuto()` helper at cmd/htmlgraph/claude.go:195 IS reachable today — but only via `htmlgraph claude --auto` (claude.go:79-82). There is no standalone subcommand that gives auto-mode users the same UX as `htmlgraph yolo`: tmux session management, --init, --track/--feature scoping, distinct help text, and parity with the yolo flag surface. 2. **Strict-mode gate is mode-blind.** isYoloFromEvent() at internal/hooks/yolo_guard.go:25 only returns true for `bypassPermissions`. Auto mode falls through, so the 5 guards that gate on ctx.IsYoloMode — worktree isolation, TDD-commit, budget, UI screenshot, diff-review — silently disengage in auto mode. (Work-item attribution and research-first guards are always-on and would fire under auto mode either way; see pretooluse.go:103-127.) 3. **Bash guard misses automode's blocklist.** bashFileWritePattern at pretooluse.go:343 catches sed/redirects/rm/mv/dd but not curl|bash, wget|sh, git push --force, tee, cp. Users without Max+ access have zero deterministic protection from these patterns. 4. **No classifier telemetry.** When automode's classifier denies a tool call, we have no record. We can't measure denial rates, tune our deterministic guards against classifier behavior, or build downstream features that react to denials. 5. **Stuck-agent retry loops.** A denied PreToolUse returns the block; the agent retries indefinitely with minor variations, burning tokens and obscuring root cause. 6. **No mode matrix docs.** Users don't know when to pick `htmlgraph yolo` vs `htmlgraph auto` vs `htmlgraph claude` vs raw `--permission-mode` flags.

Goals

  1. **Layered defense in one command** — `htmlgraph auto` launches Claude with `--permission-mode auto` AND activates the 5 yolo-gated guards (worktree, TDD-commit, budget, UI screenshot, diff-review) plus the always-on guards (work-item, research-first). Verified by: invoking the command produces both classifier blocks and HtmlGraph hook blocks for the appropriate tool patterns.
  2. **Bash guard parity with automode blocklist** — A new `bashDestructivePattern` (sibling to `bashFileWritePattern` at pretooluse.go:343) catches `curl|bash`, `wget|sh`, `git push --force` (and `-f`), `tee` to non-`.htmlgraph/` paths, and `cp` to source files; verified by table-driven tests.
  3. **Classifier-denial telemetry** — Every `PermissionDenied` hook event is recorded in `agent_events` with `event_type='permission_denied'`, the denied tool name, an input summary, and the deny reason; queryable via the dashboard.
  4. **Stuck-agent escalation** — After N consecutive `PermissionDenied` events for the same session (default 3, no successful tool_call between them), the next PreToolUse is hard-blocked with a circuit-breaker message instructing the user to intervene. Counter is a SQL aggregate against agent_events (durable, reproducible across session restarts).
  5. **User-facing mode matrix** — A reference doc explains default vs `--permission-mode plan` vs `auto` vs `bypassPermissions` vs `htmlgraph yolo` vs `htmlgraph auto`, including which guards apply in each.

Constraints

  • Plugin source-of-truth rule — hook event matrix is declared in `packages/plugin-core/manifest.json`; `plugin/hooks/hooks.json` is generated by `htmlgraph plugin build-ports`. Never hand-edit generated files.
  • Hook state rule — gates must be reproducible across session restarts; the circuit breaker counter must read from durable state (`agent_events` SQL query), not in-memory session state.
  • Hooks must not write to stderr — debug output goes to `debug.log`. Allow returns `&HookResult{Continue: true}` (Go) which serializes to `{}` JSON; deny returns `&HookResult{Decision: "block", Reason: "..."}`.
  • `event_type` CHECK enum lives in TWO synchronized locations: `internal/db/schema.go:124-129` (CreateAllTables) and `:494-497` (the `agentEventsCheckConstraintDDL` constant used by the existing `migrateAgentEventsAddCheckConstraint` rebuild at :532). Both must be updated together.
  • `HTMLGRAPH_GUARDS_OFF=1` (pretooluse.go:21) is a global kill switch — every new guard, including the destructive-bash guard and the circuit breaker, must respect it.
  • Existing index `idx_agent_events_session_ts_desc ON agent_events(session_id, timestamp DESC)` at schema.go:419 already covers the circuit-breaker's `count permission_denied for session_id since timestamp X` query — no new index needed.
  • automode requires Anthropic Max+ plan — `htmlgraph auto` must fail gracefully when the user lacks access (parse classifier-disabled error from claude CLI, or document as a pre-flight requirement); slice 1 must define what 'graceful' means.
  • Existing 5 features (feat-2d2dfc12, feat-9c14a3ef, feat-fdad254b, feat-e86dbcdb, feat-85ef5a6b) are already filed on trk-68655b2a — slices map 1:1 via `slice.feature_id` so finalize is idempotent and reuses them rather than creating duplicates (per cmd/htmlgraph/plan_finalize.go:317).

#2 Destructive bash guard — curl|bash, wget|sh, force-push, tee, cp S Low Pending
What
Add `bashDestructivePattern` regex to `internal/hooks/pretooluse.go` (alongside `bashFileWritePattern` at line 343). Match: `curl ... | (bash|sh)`, `wget ... | (bash|sh)`, `git push ... --force` (and `-f`, `--force-with-lease` to protected branches main/master), `tee` writing outside `.htmlgraph/`, `cp` to repo source files, and `chmod +x` on downloaded files. Wire a new guard `checkBashDestructiveGuard` called from `PreToolUse()` BEFORE the work-item guard so destructive intent fails fast. The guard's blocking behavior in default vs strict modes is determined by Q-bash-guard-default-mode below. Anchor regexes to command shape (e.g. `^\s*curl\b.*\|\s*(bash|sh)\b`), not substrings, per the hook state rule. Respect `HTMLGRAPH_GUARDS_OFF=1` (pretooluse.go:21).
Why
Today's bash guard catches sed/redirects/rm/mv/dd but misses pipe-from-network-to-shell and force-push — exactly the patterns automode's classifier blocks. Users on plans without automode access have zero deterministic protection. Even on Max+ with automode, the deterministic Go guard runs first and gives a faster, citable failure (file:line) before round-tripping to the classifier — and serves as a backstop if the classifier is unavailable.

#4 Consecutive-deny circuit breaker — escalate stuck-agent retry loops M Med Pending
What
Create `internal/hooks/circuit_breaker.go` with a `checkCircuitBreaker(ctx, db) (*HookResult, error)` function. It is called from `PreToolUse()` in `pretooluse.go` AFTER context resolution but BEFORE expensive guards. The check is gated on `ctx.IsYoloMode` — circuit breaker is only meaningful in strict modes (yolo/auto), and gating avoids adding a DB query to every default-mode tool call. Also respect `HTMLGRAPH_GUARDS_OFF=1` (pretooluse.go:21). Implementation: query agent_events using the existing index `idx_agent_events_session_ts_desc` (schema.go:419). Count `event_type='permission_denied'` rows for the current session_id since the most recent successful tool_call (or the session start, whichever is more recent). When count >= `circuitBreakerThreshold` (named constant, default 3), return `&HookResult{Decision: "block", Reason: "Circuit breaker tripped: N consecutive permission denials in this session. Pause and review with the user before retrying. Reset by: <answer to Q-circuit-breaker-reset>."}`. The reset mechanism is determined by Q-circuit-breaker-reset. Add a SQL helper `db.CountConsecutiveDenials(sessionID string) (int, error)` to `internal/db/event_repo.go` so the query is testable in isolation. NO new index is required — the existing `idx_agent_events_session_ts_desc` covers the query.
Why
Today a denied PreToolUse simply returns the block, and the agent retries indefinitely with minor variations — burning tokens, spinning forever, and obscuring the root cause. The circuit breaker forces a hard stop after a small number of denials so the user can intervene. Reading from `agent_events` (slice 3) keeps the gate reproducible across session restarts (per the hook state rule). Gating on `ctx.IsYoloMode` keeps default-mode latency unaffected.
