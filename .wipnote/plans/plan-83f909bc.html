Problem

wipnote's development practices (commit-what-you-close, regenerate ports after generator edits, run the real gate, link duplicates, never present a glance as proof) are currently enforced only by advisory prose in CLAUDE.md/AGENTS.md and skill text. This session produced direct evidence that prose enforcement fails: Codex left 6 items done with evaporable / never-committed artifacts; feat-e78221c7 shipped without build-ports leaving generated trees stale; partial -run-filtered test runs were presented as a passing gate; byte-identical bugs (bug-7b5339cc / bug-d67e695e) were closed independently. The existing complete-path guard (checkUncommittedSourceCompleteGate) only rejects unrelated dirty source OUTSIDE .wipnote/ - it never verifies the item's OWN artifact commit actually landed, and none of the practices are enforced identically across the three harnesses.

Goals

  1. Make completion transactional so no item can reach done with an uncommitted or never-committed artifact (compensating re-open on commit failure).
  2. Mechanically block stale generated trees from being committed or shipped.
  3. Require commit/gate provenance before completion of code-bearing items, reusing trace/lineage primitives wipnote already has.
  4. Add session-exit reconciliation that self-heals bookkeeping drift and blocks ambiguous source drift, on every harness.
  5. Detect duplicate work items at creation time.
  6. Guarantee slice-1..6 enforcement is harness-identical via the shared Go binary invoked by the per-harness hook matrix, and make weak-harness regressions visible as a tracked per-session adherence metric.

Constraints

  • Enforcement MUST live in the shared Go binary (cmd/ + internal/) invoked by the per-harness hook matrix in packages/plugin-core/manifest.json - never in per-harness bespoke logic and never as more advisory prose in CLAUDE.md/AGENTS.md (the CLAUDE.md philosophy says fixes land in the binary/hooks, never descriptive docs; advisory text is exactly what failed).
  • No new canonical state outside .wipnote/*.html artifacts; the SQLite read index is derived and not committed, so gate records and scorecards must be derivable read-index rows, not canonical .wipnote HTML.
  • Generated trees (plugin/.claude-plugin, plugin/hooks/hooks.json, packages/codex-plugin, packages/gemini-extension) are regenerated, never hand-edited; the drift gate must full-regenerate-and-diff, not checksum.
  • No new runtime dependency - stdlib only (e.g. similarity metric).
  • col.Complete flushes the canonical HTML to disk (workitem.go:258) BEFORE the artifact commit (:329); failure handling must use a compensating re-open transition, not an in-process field revert.
  • commitWipnoteArtifact is contractually non-fatal (workitem_commit.go:62-65) for YOLO/worktree callers; a strict variant must be added for the complete path without changing the existing contract.
  • The Stop hook handler explicitly never blocks (missing_events.go:114,125) and BlockExit2Error is only wired into TaskCompleted; cross-harness blocking requires an explicit harness discriminator.
