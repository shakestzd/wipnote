Problem

wipnote's causal lineage is host-locked. Hooks, the indexer, and provenance capture record paths as the agent observed them — often absolute (`/workspaces/wipnote/foo.go`, `/tmp/worktree-abc/foo.go`) or worktree- relative. When the same repo is queried from a different working directory (worktree orchestrators like our own `Agent({isolation: "worktree"})` or Conductor; same repo on a different host like Codespace → local) the lineage chain breaks: file references don't resolve, blame queries miss commits, and `wipnote lineage <id>` returns partial chains. Existing partial mitigation: the `check-host-paths` precommit gate (`cmd/wipnote/check_host_paths.go:23-28`) rejects absolute host paths in `--description`/`--body` at creation time. That gate does nothing for runtime capture by hooks/indexer/session ingest, which is where the overwhelming majority of paths enter the store. Researcher mapped 7 runtime capture surfaces plus one omitted by the initial pass (`sessions.project_dir`, surfaced by the implementation critique): 1. PreToolUse — `agent_events.tool_input` (JSON object stored as a JSON-encoded string; SINGLE-encoded, not double — corrected from initial design) 2. PostToolUse — `feature_files.file_path` (verbatim absolute) 3. SessionStart/End — `sessions.transcript_path` (out-of-scope: Claude's private store) 4. SessionStart — `<article data-project-dir="...">` in session HTML + `sessions.project_dir` SQLite column 5. WorktreeCreate/Remove — `agent_events.input_summary` free-text + `AdditionalContext` agent-facing message 6. SubagentStart — `pending_subagent_starts.cwd` 7. CLI `--files` flag — verbatim into HTML `affected_files` Existing partial helper: `internal/migrate/tracks.go:281 normalizePath` (read-time, anchor-based, wipnote-specific). The right extension point is `internal/paths/resolve.go` which already has `ResolveViaGitCommonDir` for worktree → main repo. 178 existing `.wipnote/*.html` files contain absolute paths and need a one-shot migration.

Goals

  1. **Write-time normalization** — every path captured by a hook, indexer, or CLI write is stored repo-relative against the canonical repo root resolved via `git rev-parse --git-common-dir` with walk-up fallback.
  2. **Single normalizer with cache** — one helper in `internal/paths/` used by every writer, with a process-scoped cache so `git rev-parse` is invoked at most once per unique CWD per process lifetime.
  3. **Worktree round-trip** — a feature implemented in a wipnote-managed worktree, after the worktree is removed, returns full lineage with all referenced files resolvable from the main repo root.
  4. **Migration completeness** — all 178 pre-existing `.wipnote/*.html` records plus the relevant SQLite columns are rewritten idempotently with collision detection; re-running the migration is a no-op.
  5. **No new false negatives** — `transcript_path` (Claude's private store) is explicitly out-of-scope and remains absolute; the `--allow-host-paths` escape hatch on creation flows still works; foreign-machine ingested sessions preserve their original paths.
  6. **Safe by default** — migration aborts on a dirty `.wipnote/` working tree, takes a timestamped backup, supports `--dry-run`, and has a paired `wipnote migrate restore-paths` companion for rollback.

Constraints

  • Reuse the regex from `cmd/wipnote/check_host_paths.go:23-28` as the single source of truth for what counts as a host path.
  • Worktrees: use `git rev-parse --git-common-dir` (or extend `ResolveViaGitCommonDir`), not `--show-toplevel` (which returns the worktree dir, not the canonical root). When `git rev-parse` returns `.git` (the main worktree case), fall back to walk-up resolution.
  • `transcript_path` is out-of-scope — it points to `~/.claude/projects/<encoded>/<uuid>.jsonl` which is intentionally machine-local. Leave absolute.
  • Backwards compatibility on read: lineage and blame queries must continue to resolve records written before this plan. Query callers normalize their *inputs* at lookup time as well, so the SQLite store can contain a mix of absolute (legacy) and relative (post-migration) paths without surface breakage.
  • All stored relative paths use forward-slash separators (`filepath.ToSlash`) regardless of host OS, so cross-platform queries match.
  • Symlinks in `repoRoot` and `absPath` are resolved via `filepath.EvalSymlinks` before relativization, matching the existing pattern in `internal/dbpath`.
  • One policy for outside-repo paths, defined in slice-1 and inherited by every writer: paths outside the repo are stored with an `unresolved:` prefix (not verbatim, not dropped). Rendering layers strip the prefix for display and never construct a filesystem path from the suffix.

#1 Canonical write-time path normalizer in internal/paths Add NormalizeToRepoRelative(absPath, repoRoot string) (relPath string, ok bool) M Med Promoted Approved 6 issues 1 Q
What

Add NormalizeToRepoRelative(absPath, repoRoot string) (relPath string, ok bool) to internal/paths/resolve.go. It:

  1. If absPath is already relative, return (absPath, true) unchanged.
  2. If repoRoot == "", resolve canonically: a. Try ResolveViaGitCommonDir(filepath.Dir(absPath)). b. If that returns .git (main worktree case) or empty, walk up from filepath.Dir(absPath) looking for the nearest .wipnote/ directory. c. If both fail and absPath matches HostPathPattern, return ("unresolved:" + absPath, true). Otherwise return (absPath, true) (treat as already-relative-but-exotic).
  3. Apply filepath.EvalSymlinks to both repoRoot and absPath before relativization.
  4. Compute filepath.Rel(canonicalRoot, evaledAbsPath); if the result starts with .., return ("unresolved:" + absPath, true).
  5. Convert the relative path to forward slashes via filepath.ToSlash and return (rel, true).

Add a process-scoped cache (sync.Map keyed by filepath.Dir(absPath)) so git rev-parse is invoked at most once per unique directory per process lifetime — critical because slice-2 wires this into hot-path hooks (PreToolUse/PostToolUse fire on every tool call).

Promote the hostPathPattern regex out of cmd/wipnote/ into internal/paths/hostpattern.go so both the precommit gate and the runtime normalizer reference the same definition.

Add a sibling MustNormalize(absPath, repoRoot string) string that returns the normalized path on success or the original absPath on failure, logging at debug level. Use this in hot paths where the hook shouldn't fail on a normalization edge case.

Why

Single source of truth for normalization (constraint #1). All subsequent slices wire this in at one call site each. Without a shared helper, each writer will drift its own logic and the foreign-machine / outside- repo / symlink / Windows-separator edge cases will be re-handled (and mis-handled) per call site.

#6 Idempotent migration rewriter with collision detection and dirty-tree guard New CLI subcommand wipnote migrate normalize-paths that walks L High Not Started Approved 8 issues 1 Q
What

New CLI subcommand wipnote migrate normalize-paths that walks .wipnote/ and rewrites absolute paths to repo-relative in place.

Safety preconditions (must pass before any write): a. Working tree clean — run git -C <repoRoot> status --porcelain .wipnote/; abort with a clear error if dirty. Use --allow-dirty to bypass (records audit reason). b. Collision check — for feature_files (which has UNIQUE(feature_id, file_path)), pre-scan: if two rows for the same feature would normalize to the same relative path, list them in the dry-run output. Default behavior is to MERGE (take earliest first_seen, sum sessions); --no-merge-collisions causes the migration to abort instead.

Targets (7, corrected from initial 6):

  1. agent_events.tool_input — SINGLE-encoded JSON; unmarshal once, walk path keys, re-marshal. (Initial plan said doubly-encoded; that was wrong.)
  2. feature_files.file_path (SQLite TEXT) — with collision detection.
  3. agent_events.input_summary free-text — regex-replace WorktreePath embeds.
  4. pending_subagent_starts.cwd
  5. sessions.project_dir (SQLite) — newly added; missed in initial design.
  6. <article data-project-dir> in .wipnote/sessions/*.html
  7. affected_files properties in .wipnote/{features,bugs,spikes}/*.html

Out-of-scope (documented explicitly):

  • sessions.transcript_path — Claude's private machine-local store.
  • .wipnote/.active-session — transient per-session JSON; rewritten on next SessionStart anyway.

Idempotency: re-running on already-relative records is a no-op. Achieved by skipping any value that doesn't match HostPathPattern.

Backup + dry-run + restore:

  • --backup (default on): copies touched HTML to .wipnote/.backup-<timestamp>/.
  • --dry-run: prints proposed changes and pending collisions; no writes.
  • Companion: wipnote migrate restore-paths --from .wipnote/.backup-<timestamp>/ copies backups back into place.

Print summary: N files rewritten, M values normalized, K skipped as already-relative, J marked unresolved, P collisions merged, A collisions aborted (if --no-merge-collisions).

Rendering audit: confirm wipnote feature show, wipnote blame, wipnote lineage, and the dashboard template render relative paths correctly (resolve against current repo root for display, strip unresolved: prefix). Update each call site as needed.

Why

Without migration, slices 1-5 fix the future but leave the existing 178 records broken — wipnote lineage queries still hit absolute paths in older records. Addresses goal #4.

Collision detection is critical because feature_files has a UNIQUE constraint; a naive UPDATE would silently merge rows and lose first_seen attribution.

Dirty-tree guard prevents the migration from interleaving with in-progress edits. The --backup + restore-paths pair gives a complete rollback story.
