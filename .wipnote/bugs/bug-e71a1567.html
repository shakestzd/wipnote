Read-only WAL opens can fail when -wal/-shm sidecars are absent, so converted CLI commands fail before querying. Fix: use mode=rw with PRAGMA query_only=ON, or handle and test the WAL sidecar case. (MED) cmd/wipnote/graph.go:75 — converted read commands no longer run migrations; existing cache with older schema can open but fail later with missing-table/column errors. Fix: check schema/user_version before querying and emit migration/reindex guidance, or keep migrated open path for commands that require current schema. (LOW) internal/db/open_modes.go:114 — OpenWritable is documented for an existing schema-current DB, but opening a bare path can create a new empty SQLite file. Fix: use mode=rw or os.Stat guard so missing DBs fail fast.
