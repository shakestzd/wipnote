YOLO test guard error message hardcodes both Go and Python commands instead of project-aware

Implemented In:

Description

When YOLO mode blocks a commit for failing tests in a Python (or any non-Go) project, the error message says: 'YOLO mode requires tests to pass before committing. Run: go test ./... or uv run pytest'. The message is constant and shows BOTH commands regardless of project type, confusing users in single-language projects. Source: internal/hooks/yolo_guard.go:242-243. Should detect project type from manifest files (go.mod, pyproject.toml, package.json, Cargo.toml) and emit only the relevant test command. Same project-type detection pattern is documented in the orchestrator system prompt.
