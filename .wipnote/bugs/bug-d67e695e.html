Description

Dashboard activity feed renders each Gemini user prompt twice: one row from buildEventTreeOtel (interaction span, iconed, real tool count) and one from buildEventTreeOtelLogFallback (zero-ts log, often 0 tools). The fallback's NOT EXISTS filter compares ts_micros directly, but Gemini user_prompt logs have ts_micros=0 (real ts in attrs_json), so NOT EXISTS passes. Secondary defense at filterAnchorsAgainstInteractionSpans (api_tree.go:318-320) short-circuits zero-ts anchors as 'keep' instead of filtering them, letting the duplicate through.
