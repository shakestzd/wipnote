Description

Manager.Start now rejects invalid agent (not in {claude,codex,gemini,yolo}) and invalid work_item (not matching ^[a-zA-Z0-9_-]+$) with a typed error. handleTerminalStart maps ALL Start errors to http.StatusServiceUnavailable, so validation failures come back as 503 when they should be 400 BadRequest. Fix: differentiate invalid-input errors from runtime errors (ttyd-missing, fork-failure) — either via a sentinel errors.Is check or by moving input validation into the handler before calling Start. Surfaced during /roborev-fix Playwright validation.
