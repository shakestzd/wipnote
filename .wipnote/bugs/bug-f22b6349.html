Description

Every htmlgraph CLI command emits: '[htmlgraph] WARNING: legacy SQLite file at .htmlgraph/htmlgraph.db (0 MB) is unused — DB now lives in the user cache dir.' This bloats every agent transcript and adds noise that crowds out signal. Two acceptable fixes: (a) auto-delete the legacy file on first detection (it is empty / vestigial; the user is told to delete it anyway) and emit the warning at most once via .htmlgraph/.legacy-warned marker, (b) gate the warning behind a once-per-day check via mtime on a state file under the cache dir. Preferred: (a), with the auto-delete behavior gated on the file being zero-byte and the message logged at INFO level rather than WARNING. The warning has shown up in every dispatched agent transcript for the past N sessions and is no longer actionable.
