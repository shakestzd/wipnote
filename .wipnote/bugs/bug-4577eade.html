Description

Two cross-harness behaviors deserve being documented so future contributors don't get confused. (1) wipnote codex mirrors agent TOMLs into ~/.codex/agents/wipnote-*.toml on every launch but never cleans them up on exit — by design, since the user may want them present for ad-hoc codex sessions outside the wipnote launcher. (2) Per https://code.claude.com/docs/en/sub-agents, Claude Code plugin subagents silently drop the hooks, mcpServers, and permissionMode frontmatter fields. wipnote doesn't currently use these, but anyone adding agent fields needs to know.
