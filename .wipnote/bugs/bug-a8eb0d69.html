Worktree gitdir path breaks cross-host (macOS to Linux Codespace) without manual repair

Done Medium Priority

Part Of:

  • trk-cd61bbae

Description

A git worktree created on macOS stores an absolute gitdir path like /Users//project/.git/worktrees/. When the same repo is opened on a Linux Codespace at /workspaces/, the worktree .git file still points at the macOS path and every git command inside the worktree fails: 'fatal: not a git repository: /.git/worktrees/'. I had to manually rewrite both the worktree's .git file and the main repo's .git/worktrees//gitdir file to unblock /htmlgraph:execute. The slice-7 subagent scaffolded a RepairGitdir helper in internal/worktree/worktree.go but it was never wired to any caller and I reverted it as out-of-scope. Fix: wire RepairGitdir (or equivalent) into the CLI entry path so htmlgraph in a worktree self-heals stale paths on first run, emitting a one-line log. See also stash@{0} which may preserve a partial fix. Surfaced at the very start of /htmlgraph:execute trk-cd61bbae.
