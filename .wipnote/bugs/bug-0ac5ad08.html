Description

The per-session OTel collector buffers events in memory and only flushes the session NDJSON file at clean shutdown. On host sleep, devcontainer disconnect, or kill -9 the in-flight events are lost. Evidence (2026-04-27): the running yolo session's events file at .htmlgraph/sessions/019dce916bc529b8b5c635533eab is 96 bytes, last touched 06:52:11.78 — just the session-registration stub. The collector process (PID 66018) was alive for 32 minutes listening on 127.0.0.1:42045 but find shows zero NDJSON files for that session ID anywhere on disk. By contrast, healthy yesterday's sessions hold 800 KB to 5.9 MB of events.ndjson per session. Result: observability gap during exactly the long-running yolo / planning sessions we'd most want to inspect after the fact (e.g., to root-cause the recording bug that forced HTMLGRAPH_GUARDS_OFF=1 on the feat-9b767422 commits). Fix shape: incremental fsync — flush after every N events or every K seconds, whichever comes first; reopen the file in append mode on collector restart so a replacement collector can extend the same session log instead of starting empty. Surfaced while validating feat-9b767422.
