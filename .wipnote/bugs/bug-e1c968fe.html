Description

Symptom: a Claude Code session opened in /.claude/worktrees/trk-cd61bbae (Linux devcontainer) had CWD showing '/.claude/worktrees/trk-cd61bbae' and the worktree's .git file contained 'gitdir: /.git/worktrees/trk-cd61bbae' — a macOS path that does not exist in the container. All git commands failed until the .git file was manually rewritten. This is the same class as the existing memory about settings.local.json cross-machine path drift. Fix options: (a) on /htmlgraph:execute entry (or session_start hook), detect mismatch between worktree .git gitdir and the current repo root and rewrite before any git command runs; (b) when creating worktrees via htmlgraph, use a relative gitdir where supported, or regenerate on session_start; (c) move worktrees out of .claude/worktrees into an env-local location (e.g. ~/.htmlgraph/worktrees//) that is not shared across hosts. Acceptance: fresh Claude Code session in a worktree created on the other host works immediately, no manual .git repair.
