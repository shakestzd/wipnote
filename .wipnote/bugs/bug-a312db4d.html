Description

Multiple SQLite DB paths in use across the codebase: serve_child.go opens .htmlgraph/.db/htmlgraph.db while a 77MB orphan sits at .htmlgraph/htmlgraph.db. The .db/ subdir was a devcontainer-specific workaround for WAL-on-virtiofs that doesn't help end users (their projects also live on virtiofs/osxfs in Docker Desktop, devcontainers, Codespaces). Durable fix: relocate the SQLite read-index out of the project tree entirely into os.UserCacheDir()/htmlgraph//htmlgraph.db. SQLite is a derived index per the architecture (HTML+NDJSON are canonical), so cache-dir is the correct semantic location. Single helper storage.CanonicalDBPath() owns resolution; HTMLGRAPH_DB_PATH env var allows override. Static + runtime guards prevent regression. Drop the htmlgraph-db named volume from .devcontainer/devcontainer.json — it becomes dead config. Acceptance Criteria All sql.Open callers route through storage.CanonicalDBPath. Static unit test fails if any non-storage Go file contains literal htmlgraph.db. Runtime guard warns on legacy project-local DB files. devcontainer named volume removed. Quality gates pass.
