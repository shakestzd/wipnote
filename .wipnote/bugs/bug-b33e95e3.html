checkProjectDivergence: '.' short-circuit disables cross-project protection (roborev #1693)

Done High Priority

Caused By:

  • feat-bd410702

Part Of:

  • trk-08dcbb33

Description

Roborev #1693 caught regression in commit 1aa67324d. The 'sess.ProjectDir == "."' short-circuit was too aggressive: any session whose stored project_dir happens to be "." would bypass the divergence check entirely, allowing write tools to run against a different wipnote project without being blocked. DB shows 8/136 sessions have "." (anomalies, not the norm), so the protection should stay. Proper fix: remove the short-circuit and trust canonicalRepoRoot (which already calls filepath.Abs for relative paths). Add regression test with sess.ProjectDir="." + event.CWD in another project that MUST block.
