Description

The dashboard polls fetchStats() every 30 seconds via the setInterval at cmd/htmlgraph/dashboard/js/app.js:1252-1254, but fetchSessions() is only called once when the user switches to the Sessions view (app.js:27). After that, the sessions list is frozen indefinitely until the user manually re-navigates. Symptom: an active session shows correct LIVE status but the message count and activity bars are stale by minutes or hours. Fix: add fetchSessions() to the existing setInterval (or a separate one with tighter cadence — 10-15s when an active session is in view). Three-line change. Verify by opening the Sessions view, sending a few messages from another Claude Code session, and confirming the message count updates without manual refresh.
