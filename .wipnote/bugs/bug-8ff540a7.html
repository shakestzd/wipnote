Done High Priority

Part Of:

Description

Mirror of GitHub issue #90 (https://github.com/shakestzd/htmlgraph/issues/90). When the parent Claude Code session is in YOLO mode (permission prompts disabled, full quality enforcement), subagents spawned via Agent({isolation:'worktree'}) do NOT inherit YOLO mode. They run with the default permission posture — combined with htmlgraph hooks that gate Edit/Write on YOLO mode, subagents are blocked outright. Subagent burns its tool budget bouncing off the permission gate. Same family as bug-0413c3c4 (#87, attribution claim session-ID mismatch) and bug-730c0f19 (#88, commit-to-main isolation). The pattern is identical: hook check uses sub-agent's session ID directly instead of walking the parent chain to find the orchestrator's posture. Suggested fix (from issue, in order of decreasing invasiveness): 1. Propagate YOLO mode through Agent subagent spawns (env/config injection) 2. Make hook YOLO-detection inheritable — walk up to parent session record in agent_events via parent_session_id; treat subagent as YOLO if parent is. SAME PATTERN AS #87 FIX (parent-chain walk). RECOMMENDED. 3. Document the limitation in /htmlgraph:execute Recommended: option 2. Reuses the parent-session-chain walker introduced in plan-ff425b9f slice 1. This bug becomes a small slice 3 of that plan: apply the same getSessionAndParent walk pattern at the YOLO-check site.
