dbpath: hard-coded /tmp fallback breaks on non-Unix when TMPDIR unset

Part Of:

Relates To:

Description

Roborev #1507 (1 MED): internal/storage/dbpath.go:78 — candidateRoots falls back to literal '/tmp' when TMPDIR is unset. On non-Unix platforms (especially Windows where ProbeFsType reports all paths as WAL-safe), this can select an unexpected rooted '\tmp\wipnote\...' path instead of the platform cache directory. Fix: use os.TempDir() for the temp fallback, or only add the hard-coded '/tmp' candidate on Unix-like platforms.
