Root dashboard fetches /api/terminal/sessions which 404s in multi-project serve

Done Medium Priority

Part Of:

  • trk-cd61bbae

Description

When htmlgraph serve runs in multi-project (parent/child) mode, the root dashboard at / loads the same app.js that the per-project dashboard at /p// does. app.js polls /api/terminal/sessions every 4s and on DOMContentLoaded, but the terminal routes are only registered inside the per-project mux (serve.go:113-116). At /, each poll spams the console with 'Failed to load resource: 404 Not Found @ /api/terminal/sessions' and the multi-pane UI silently renders nothing. Either the root dashboard should not attempt terminal fetches when the per-project prefix is absent (detect via buildProjectUrl context), or serve should register a /api/terminal/* stub at the parent that 501s cleanly so the console isn't noisy. Surfaced during /roborev-fix Playwright validation.
