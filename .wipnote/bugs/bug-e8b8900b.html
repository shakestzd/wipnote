Description

Symptom: running two parallel Claude Code sessions in two terminal windows in the same project directory produces one combined session in the htmlgraph dashboard after ingest. Messages from both conversations appear interleaved in a single session_id entry. Two possible root causes: 1. Claude Code writes both transcripts to the same JSONL file in ~/.claude/projects// — this would be a Claude Code behavior (shared session_id across parallel terminals in the same project) that htmlgraph faithfully mirrors 2. Claude Code writes two separate JSONL files but htmlgraph's ingest merges them under a shared session_id — this would be a real htmlgraph ingestion bug Investigation needed: - Enumerate recent *.jsonl files in ~/.claude/projects/-Users-shakes-DevProjects-htmlgraph/ to see if there are 1 or 2 files for the parallel sessions - Query the sessions table for recent session_ids and their source (hook vs ingest) and message counts - Inspect a suspected merged session to see which session_ids its messages carry at the storage layer - If there are 2 JSONL files but 1 session row: ingest bug (dedup/merge logic) - If there's 1 JSONL file with both conversations: Claude Code behavior that htmlgraph needs a workaround for (e.g. split by user_query/turn detection) Impact: violates the core observability guarantee. If parallel sessions can't be distinguished, the 'see what your agents do' value prop is compromised for any user who runs multiple terminals.
