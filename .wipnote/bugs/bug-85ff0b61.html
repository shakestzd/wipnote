Indexer logs every orphan-session skip on every scan pass (log spam)

Todo Medium Priority

Part Of:

  • trk-0677c709

Description

The indexer logs one 'skipping orphan session dir (age=Xd, last-write=...) — no DB row, stale and quiescent' line per orphan on EVERY scan pass. With a volatile tmpfs read-index DB (wiped on restart) and durable session transcript dirs, dozens of 3-4d-old orphans get re-logged each indexer cycle, flooding serve output. Skip decision itself is correct (do not re-ingest stale quiescent orphans); the defect is log hygiene: should log-once, aggregate a count, or drop to debug level rather than per-orphan per-pass at info. Repro: tmpfs DB reset + many old session dirs + running wipnote serve indexer loop, observed 2026-05-15 18:35.
