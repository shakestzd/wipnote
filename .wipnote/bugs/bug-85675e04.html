Description

The pre-commit hook at .githooks/pre-commit runs go build + go vet + go test ./... on every commit. In the ARM64 Codespace, the cmd/htmlgraph package alone takes ~67s; the full suite is 5-8 minutes. During /htmlgraph:execute three dispatched subagents hung on their final git commit waiting for this hook, and the agent output truncated before completion — leaving uncommitted work on main that I had to commit manually. Fix options: (1) scope test run to packages with staged Go files via git diff --cached; (2) run tests in -short mode for commit hooks and reserve full suite for CI; (3) offload test run to a background CI job and only block on build+vet.
