Description

Many cmd/htmlgraph/*_test.go files create temporary project roots via t.TempDir() and then exercise code paths that resolve storage.CanonicalDBPath(projectDir). CanonicalDBPath hashes the absolute project path and returns /htmlgraph//htmlgraph.db, so each unique tempdir produces a fresh subdirectory under the user's real cache. Across a single 'go test ./cmd/htmlgraph/...' run this can create hundreds of entries. Evidence (2026-04-27): the ~/.cache/htmlgraph cache regrew to 6,022 subdirs / 2.5 GB, with 2,012 new subdirs / 834 MiB added in the four hours immediately after eviction was wired up — the new opportunistic prune fired at 10:21 but didn't evict anything because entries were <90 days and the total dipped under the 1 GiB threshold between bursts. Defeats the eviction policy added in feat-9b767422 entirely, and is the same shape that caused the original 17,843-subdir / 7.3 GB explosion (spk-dfb051a3). Tests that touch CanonicalDBPath should either set HTMLGRAPH_DB_PATH to t.TempDir()/htmlgraph.db (workitem_test, projects_test, plan_feedback_cmd_test, cleanup_test already do) or share a single per-package cache override via TestMain. Surfaced while validating feat-9b767422.
