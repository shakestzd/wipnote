Description

cmd/htmlgraph/hook.go hookSubcmd constructors contain 'database, err := db.Open(hooks.DBPath(projectDir)); if err != nil { return fallback, nil }'. When Open fails (e.g., PRAGMA journal_mode=DELETE can't acquire the exclusive lock needed to switch out of WAL mode), the hook returns a fallback result with no log line. Hid the NewWriter-forces-WAL regression for hours: user-prompt hooks completed in 3-4ms with zero events written and zero errors logged. Fix: before returning fallback, debugLog '[error] handler=%s session=%s: db.Open failed: %v'. Apply in every hookSubcmd / hookSubcmdWithProject / hookSubcmdWithTool that calls db.Open. Files: cmd/htmlgraph/hook.go.
