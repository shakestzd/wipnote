Activity feed: nest <task-notification> wake-turns under originating tool row

Todo Medium Priority

Caused By:

  • bug-44ee9dd8

Part Of:

  • trk-0677c709

Description

When a background task completes, the harness re-invokes the agent with a system message. The feed renders this resumption turn as a top-level row whose title is the raw ... ... XML. It should instead nest under the row containing the originating background tool call (correlated by tool-use-id), and never display raw notification XML as a row title.
