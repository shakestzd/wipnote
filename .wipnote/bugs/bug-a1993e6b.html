Description

checkProjectDivergence in internal/hooks/pretooluse.go:661 compares ResolveProjectDir(event.CWD) against sess.ProjectDir. When a session is started in main repo but events fire from a linked worktree (or vice versa), the two values can differ even though they are the SAME LOGICAL PROJECT (worktree + main share a .git common dir). The mismatch triggers a hard block on every Write/Edit with the message 'CWD has changed to a different project'. Wedged the YOLO session attempting to start feat-5ddde9d7 inside the trk-08dcbb33 worktree on 2026-05-13 — exactly the failure mode that motivated the auto-commit work in feat-fdf99806. Fix: treat linked worktrees of the same repo as the same project by normalizing both paths through paths.ResolveViaGitCommonDir before declaring divergence.
