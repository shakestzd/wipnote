Description

Visible race: subagent-emitted tool spans (Read/Edit/Bash) reach the dashboard BEFORE their parent Agent/subagent_invocation span does (because Agent has long duration and OTel spans emit on END). _indexSpans sees parent_span pointing at nothing, synthesizes a pending root, and renders them at turn root. Moments later the Agent span arrives, re-render nests them correctly. User perceives a flash / lag in nesting. Fix options: (1) grace-period orphan queue — stash orphans for ~5-10s, reconcile on each new span batch, only synthesize roots after timeout; (2) optimistic subagent reparenting — when a same-trace Agent span has started but not ended and its start_time precedes the orphan, provisionally reparent under it. Dependent on F4/F1/F3/F2 polish pass landing first (they also touch event-tree.js indexing).
