Description

GitHub issue #85 (https://github.com/shakestzd/htmlgraph/issues/85). Finalize Plan button fails with HTTP 400 'not all sections approved' despite UI showing all N sections approved. Root cause per reporter: plan_feedback writer stores value='approved' for approve actions, but the count query filters WHERE action='approve' AND value='True' — mismatch returns 0, so a fully-approved plan never finalizes. Affects htmlgraph 0.57.0 (current). Bonus UX bug: dashboard JS surfaces all non-network finalize failures as 'Could not reach server. Is htmlgraph serve running?' (misleading) at js/app.js ~line 3587. Reporter included full repro: curl POST returns 400 with body 'not all sections approved'; curl GET returns approved=0 / total=11. DB inspection confirmed 11 rows in plan_feedback with value='approved' (both old and freshly-written rows on 0.57.0).
