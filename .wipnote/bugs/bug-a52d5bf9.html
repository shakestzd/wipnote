The display-side fix (bug filed same day) adds a WHERE s.project_dir = ? filter to sessionsHandler which hides these rows from the per-project dashboard. But the underlying data bug remains: some ingest path writes session rows to the current project's .htmlgraph/htmlgraph.db with project_dir='', and that happens on every project that ever runs htmlgraph. Likely source: passive session ingestion from Claude Code's ~/.claude/projects//*.jsonl transcripts. When htmlgraph first runs in a project, it may scan the transcript directory and populate its local sessions table. If it can't determine which project a given transcript belongs to, it writes empty project_dir and the row becomes cross-project pollution. Needs investigation: 1. Find the ingest path (grep cmd/htmlgraph/ for 'claude/projects' or 'jsonl' parsing) 2. Determine whether the ingest should (a) attribute transcripts to their real source project, (b) skip unattributed transcripts, (c) write them to a dedicated 'claude-code' bucket, or (d) not run at all when no attribution is possible 3.
