Description

claude_env.go:49 uses addIfUnset for OTEL_EXPORTER_OTLP_ENDPOINT. When the launcher's parent env already has that variable set (inherited from a prior htmlgraph claude session whose computed port differs from this session's hash), addIfUnset keeps the stale value. Claude Code then sends OTLP to a port no receiver is bound to and spans are silently dropped. Symptom: dashboard OTel panels stay empty with no hint why. Durable fix: (1) change addIfUnset to setOrReplaceEnv for OTEL_EXPORTER_OTLP_ENDPOINT so the launcher-computed endpoint always wins. Users who want an external endpoint override should use HTMLGRAPH_OTEL_HTTP_PORT / HTMLGRAPH_OTEL_BIND which LoadConfigFromEnv already honors. (2) Add a ~1s TCP probe at launcher start; if the computed host:port is not reachable, log a one-line stderr warning naming the endpoint. Files: cmd/htmlgraph/claude_env.go (primary), cmd/htmlgraph/claude_env_test.go (assertion that pre-existing env gets overridden). Symmetry note: line 29 already does setOrReplaceEnv for HTMLGRAPH_PROJECT_DIR — overriding the paired endpoint is consistent.
