CLI commands don't show resolved project dir when env vars override CWD

Done Medium Priority

Part Of:

  • trk-97f85b3b

Implemented In:

  • session 6206b18b-775e-4c99-8f0c-c6c61c8c954d

Description

When HTMLGRAPH_PROJECT_DIR or CLAUDE_PROJECT_DIR is set (normal state inside a Claude Code session), htmlgraph CLI commands silently operate on that project regardless of CWD. If the user cd's into a different project and runs a command, the output is about the env-var project without any indication. Surfaced during plan-43659c92 execution: ran 'htmlgraph migrate sessions --dry-run' from planbook expecting it to check planbook, but it silently checked htmlgraph and reported 'No orphan sessions'. Fix: project-level mutation commands (migrate, sweep) should print a one-line header showing the resolved project dir when it differs from CWD.
