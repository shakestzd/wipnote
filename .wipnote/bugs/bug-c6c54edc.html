Description

Mirror of GitHub issue #92 (https://github.com/shakestzd/htmlgraph/issues/92). When /htmlgraph:execute dispatches parallel sub-agents, agents can pause when they exhaust tool-use budget. The Agent runtime returns 'agentId: ' with 'worktreePath: {continue: true}' and the hint 'use SendMessage with to: to continue this agent.' But SendMessage is NOT a loaded tool in Claude Code — referenced in Agent docstrings but not in deferred-tool list. ToolSearch returns no match. Paused agents' worktrees clean up before commits land. In-progress work lost. Skill has no fallback. Suggested fixes from issue: 1. Document SendMessage as hard prerequisite for /htmlgraph:execute; verify before dispatching. 2. Update skill to detect 'worktreePath: continue:true' pause shape; re-dispatch fresh agent with prior context, OR surface clear failure to orchestrator. 3. Reduce default prompt size — strip inline TDD boilerplate; link to shared TDD protocol skill. 4. Add --no-tdd or --minimal-prompt mode for slices where codebase has test coverage and agent only extends it. Repro: /htmlgraph:execute on track with 7+ slices, 3 parallel htmlgraph:sonnet-coder agents with TDD prompts; agents run ~50 tool calls and pause near commit; SendMessage unavailable; worktrees cleaned without commits.
