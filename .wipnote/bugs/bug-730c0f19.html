Dispatched sub-agents commit directly to main instead of isolated branch

Description

Mirror of GitHub issue #88 (https://github.com/shakestzd/htmlgraph/issues/88). When orchestrator dispatches sub-agents via Agent(isolation:worktree, ...), some sub-agents land commits directly on main rather than on isolated worktree branches as documented. Two of N sub-agents in the cited dispatch had this happen; quality gates passed by luck — broken main is the failure mode. Suspicion (from issue): isolation:worktree is being interpreted as 'reuse the parent's working directory' when the parent is itself inside a worktree. The parent orchestrator was running inside a track worktree (e.g., .claude/worktrees/1-0-ui-completeness-close-trk-9a3c622d on branch trk-9a3c622d); when sub-agents spawned with isolation:worktree, no new worktrees were created (git worktree list showed no sub-agent worktrees), and the agents wrote/committed in the parent's wd, which after a checkout had ended up on main. Verbatim from agent reports: - 'Branch: trk-9a3c622d, Commit: df1b7c5' — but commit actually landed on main - 'Branch: main (commit a6c2119)' — explicitly main Affected commits visible in git log main: - df1b7c5 feat(web): wire Config page to /api/config (feat-73190d78) - a6c2119 fix(web): sidebar counts use real applications + ui_phase (feat-193d40b1) Impact: the /htmlgraph:execute model assumes per-agent isolation for independent review/merge/revert. Direct commits to main eliminate that quarantine. Same path could leave main broken if a sub-agent fails quality gates after committing. Note: isolation:worktree is a Claude Code Agent tool primitive, not htmlgraph code. The fix may require: (a) HtmlGraph-side wrapper or PreToolUse guard that detects when a sub-agent is about to git commit on main and refuses (or rewrites to a per-agent branch) (b) Documentation update warning that isolation:worktree is unreliable when the parent is in a worktree (c) Upstream Claude Code report Recommended: (a) — a PreToolUse guard at hook level that refuses git commit calls from sub-agents when on main without an explicit per-agent branch. Defense-in-depth at htmlgraph's layer, not dependent on Claude Code's worktree creation behavior.
