Description

Resolved by bug-a312db4d. The SQLite read-index now lives in the host's OS user cache directory (resolved via storage.CanonicalDBPath, keyed by sha256(projectDir)) — always on a filesystem that supports WAL/SHM mmap (ext4, APFS, etc). The busy_timeout fragility was a symptom of running WAL on virtiofs; with the canonical DB outside the project tree, that class of failure is gone for every user, not just our devcontainer.
