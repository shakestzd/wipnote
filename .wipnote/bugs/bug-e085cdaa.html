Fix: Race condition in Manager.Start — ttyd socket not bound before returning port

Done Medium Priority

Implemented In:

  • session c89a0910-ba83-4833-9dee-5364fab28c1a

Part Of:

  • trk-d8aef97a

Description

Manager.Start returns port immediately after cmd.Start() forks, but ttyd hasn't bound the listening socket yet. Frontend sets iframe.src → Chrome gets ERR_CONNECTION_REFUSED → caches that state → user sees broken iframe even after ttyd is actually up. Solution: poll 127.0.0.1: with TCP dial until it accepts or deadline expires (3s timeout). If deadline expires, kill the process and return error.
