Description

Follow-up to bug-0beaa018 (commit 88f223e1). The setOrReplaceEnv fix is correct but not sufficient. buildClaudeLaunchEnv(htmlgraphProjectDir) receives an empty string when the caller is not in a worktree (common case). Only sets HTMLGRAPH_PROJECT_DIR env if arg is non-empty, then passes same empty string to otelEndpointFromEnv. LoadConfigFromEnv: no HTMLGRAPH_OTEL_HTTP_PORT, no projectDir, no HTMLGRAPH_PROJECT_DIR env -> PortForProject('') returns base 4318. Serve-child computes PortForProject from its own cwd and binds the hashed port (e.g. 4798). Result: OTLP client sends to 4318, receiver on 4798, spans dropped silently. Fix: in claude_env.go buildClaudeLaunchEnv, derive projectDir from os.Getwd() when htmlgraphProjectDir arg is empty, and unconditionally set HTMLGRAPH_PROJECT_DIR in env before computing endpoint. Add regression test. Files: cmd/htmlgraph/claude_env.go + claude_env_test.go.
