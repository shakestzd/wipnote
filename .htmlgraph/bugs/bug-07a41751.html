Description

On every htmlgraph CLI invocation, an INFO line prints to stderr like: [htmlgraph] INFO: legacy SQLite file at .htmlgraph/htmlgraph.db (2.7 MB) is unused — DB now lives in the user cache dir. You can delete: This pollutes stderr forever for any project that has a pre-cache-migration .htmlgraph/htmlgraph.db until the user manually rms it. Breaks scripted/piped CLI usage (e.g. htmlgraph plan show | head -100). Root cause: cmd/htmlgraph/main.go:340 calls storage.WarnIfLegacyDBPresent — info-only, never removes. A separate function storage.CleanLegacyDBIfSafe (internal/storage/dbpath.go:144) is fully written and tested to safely os.Remove the legacy file when the canonical cache DB exists and is non-empty, but it is dead code in production — only tests reference it. Fix: swap the call at main.go:340 from WarnIfLegacyDBPresent to CleanLegacyDBIfSafe. Then delete the now-orphan WarnIfLegacyDBPresent function (and any tests that target it specifically). Existing TestCleanLegacyDBIfSafe_* tests already cover the new behavior including the canonical-missing/empty cases (warns instead of removing). Severity: low (cosmetic/UX) but high friction for piped-output users.
