The message is a one-time deprecation/migration nudge intended for dogfooding/devcontainer use during the SQLite-cache-relocation migration — it should not fire repeatedly in production where users have nothing to do about it. Fix candidates: (a) gate the warning behind a 'dev' build tag or env var; (b) make it a one-shot stamp (write a marker once shown, suppress thereafter); (c) drop the message entirely in production builds. Investigation: find the print site (likely internal/db/ or internal/paths/) and the gate that should have suppressed it.
