Description

plan-14772c97 (Agent Teams plugin integration, finalized 2026-04-11) has only a .yaml file in .htmlgraph/plans/; no corresponding .html canonical file exists. AGENTS.md states .htmlgraph/*.html is the canonical store. The missing HTML means the plan is invisible to htmlgraph plan list (HTML-backed), htmlgraph plan show, and htmlgraph plan set-status — all return 'plan file not found'. It does appear in htmlgraph plan list-yaml. Likely an artifact of the HTML-canonical migration (plan-3bbc313f, done 2026-05-01): pre-migration plans that weren't actively touched at migration time may have been missed by the regeneration sweep. Surfaced today during the agent-teams plan integration. plan-2c7f1b7d explicitly supersedes plan-14772c97 in its design.constraints, but the CLI couldn't mark plan-14772c97 as completed because of the missing HTML. Cohort scope: unknown how many other pre-migration plans share this state. A scan of .htmlgraph/plans/*.yaml without paired .html would identify the full cohort. Fix candidates: (a) one-shot script that scans .htmlgraph/plans/*.yaml and regenerates missing .html via the existing commitPlanChange path; (b) extend htmlgraph reindex to regenerate missing canonical artifacts, not just rebuild SQLite indexes; (c) make set-status auto-regenerate the HTML on first call. (a) is the cleanest one-shot fix for the cohort.
