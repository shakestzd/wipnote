Dashboard Plans tab strips project context — shows 0 plans for active project

Todo Medium Priority

Part Of:

  • trk-cb36e595

Description

Symptom: navigating to a project's Plans tab in the dashboard renders 'Plans 0' even when the project has many plans in its index. Top-bar stats also show --/--/--/-- instead of project counts. Root cause (partial): server returns HTTP 301 from /p//plans (and /p//plans/) to /plans/, stripping the project context. The SPA's data layer then queries a global plans endpoint instead of the project-scoped one, returning empty. API confirms data is available: /p/46c1de7d/api/plans returns plan-72685272 with full detail and 17 other plans. /api/plans (no project prefix) returns 404, which is what /plans/ likely tries to query. Reproduce: htmlgraph serve --port 8088 --bind 0.0.0.0; navigate to http://localhost:8088/p//, click Plans tab; observe Plans 0. Fix candidates: (a) make /p//plans not redirect — render the project-scoped Plans view directly; (b) preserve project context in the SPA's URL state so client-side data fetches use /p//api/plans; (c) make the global /api/plans aggregate across projects when query param ?project= is set. Discovered while attempting to review plan-72685272 (OODA pilot) per the htmlgraph:plan workflow Step 5.
