Description

Today there is no CLI command that, given a source file path, lists which features and bugs have touched it. The data exists: internal/db/feature_files_repo.go tracks file-to-feature attribution and git_commits ties commits to features. The missing piece is the reverse-lookup surface. Output: human-readable text default, plus --format json and --format markdown. Walk: → feature_files rows → features → contained_in track edges → rollup with per-feature line-change counts and dominant-track summary. This is the foundational primitive for the partner command 'htmlgraph code-areas' — once both exist, file→track attribution becomes a live derivation rather than a manually-maintained doc.
