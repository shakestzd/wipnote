Description

Two pieces. (1) On SessionEnd hook, read .htmlgraph/sessions//.collector-pid (written by slice 3), send SIGTERM, wait up to 3s for clean drain, fall back to SIGKILL if needed. This is Q3 primary layer; the collector own idle-timeout (slice 2) is the tertiary safety net; the launcher deferred SIGTERM (slice 3) is the secondary fallback. (2) Retention job in a new internal/otel/retention/ package: on serve startup and every 24h thereafter, walk session directories, detect sessions whose sessions.status = done is older than HTMLGRAPH_SESSION_RETAIN_DAYS (default 30), archive events.ndjson into .htmlgraph/archive//.tar.gz using stdlib archive/tar + compress/gzip (not .tar.zst — zstd is not a current dependency), and delete the live session dir (indexer-populated SQLite rows age out via existing cleanup commands, unchanged). Dry-run mode via HTMLGRAPH_RETENTION_DRYRUN=1 logs intended actions without moving files. Also adds htmlgraph session restore that extracts .tar.gz back into .htmlgraph/sessions/ so the indexer picks it up on next replay. ## Why Without SessionEnd SIGTERM, collectors either persist past session lifetime (waiting for a launcher SIGTERM that may never fire after launcher SIGKILL), or get reaped by OS cleanup with spans still in their batch buffer — losing them. Without retention, .htmlgraph/sessions/ grows unbounded over months. Using stdlib gzip (rather than zstd) keeps the retention pipeline dependency-free — compression ratio on NDJSON log files is secondary to minimizing new dependencies at this scale. Restoring an archived session is supported so lineage queries over historical transcripts still work. This slice closes the correctness gaps named in Q3 (primary + secondary drain layers) and the operational gaps noted in the design discussion. ## Done When - SessionEnd hook reads .collector-pid, sends SIGTERM, waits up to 3s, falls back to SIGKILL; the collector flush completes and the final NDJSON batch has no torn last line. - Retention job runs at serve boot and on a 24h timer; HTMLGRAPH_RETENTION_DRYRUN=1 logs intended archive/delete actions without moving files. - Retention archives use .tar.gz format via stdlib archive/tar + compress/gzip; no new go.mod dependency is introduced. - A session archived to .htmlgraph/archive/ is restorable via htmlgraph session restore ; the indexer picks up the restored NDJSON on next replay without duplicating rows.
