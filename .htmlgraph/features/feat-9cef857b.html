Description

Add a new PreToolUse guard at `internal/hooks/worktree_commit_guard.go` that intercepts Bash tool calls matching `git commit` from sub-agent sessions and refuses if the working directory's branch is `main` or `master`, unless an explicit per-agent branch convention is in effect. **Implementation:** 1. **Detect the Bash tool call.** PreToolUse fires for every tool. Filter for `tool_name == "Bash"` AND `tool_input["command"]` matches a regex like `^\s*git\s+commit\b` (case-insensitive). Fall through for non-commit Bash calls. 2. **Identify sub-agent sessions.** Use slice 1's `getSessionAndParent` to determine if the current session has a parent (i.e., is a sub-agent). If the current session is the orchestrator (no parent), allow normally -- the orchestrator commits on main intentionally. 3. **Resolve current branch.** Run `git -C rev-parse --abbrev-ref HEAD` from the hook to get the branch. If it returns `main` or `master` AND the session has a parent: refuse with structured error. 4. **Per-agent branch convention** (escape hatch): if the branch matches `worktree-agent-*` or `subagent-*` or has the env var `HTMLGRAPH_AGENT_BRANCH=1` set in the hook's spawned env, allow even on main. This lets dispatch tooling explicitly mark "this is a per-agent branch I created intentionally" without forcing branch-name matching. 5. **Error message structure:** the refusal must include (a) which session is blocked, (b) what branch it's on, (c) suggested remediation: "create a per-agent branch first: `git checkout -b worktree-agent-` then commit and merge to the track branch via the orchestrator." Include the parent-session ID for cross-reference. 6. **Wire the guard into pretooluse.go's dispatch.** Register it AFTER slice 1's claim-check enhancement (deps: [1]) so the parent-chain logic is available. Keep the guard small and isolated in its own file.
