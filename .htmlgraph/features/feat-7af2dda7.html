Description

Direction 1 from the design research in this session. Replace the 6-color Tailwind-500 graph palette (#22c55e, #3b82f6, #ef4444, #f59e0b, #8b5cf6, #06b6d4) with one that uses existing CSS design tokens so the graph looks native to htmlgraph. Concrete color assignments: Track -> var(--accent) (chartreuse #CDFF00 dark / olive #4a6e00 light — the brand) Feature -> var(--text-secondary) (high-contrast neutral, dominant by mass) Plan -> var(--text-muted) (lower-contrast neutral, dashed border to differentiate from feature) Bug -> var(--status-blocked) (semantic red, already in the design system) Spike -> var(--priority-high) (semantic amber, already in the design system) Session -> var(--status-ip) (semantic blue, already in the design system) Universal improvements that apply regardless of palette choice: - Contrast-aware text color via YIQ luminance: pick var(--text-primary) or its inverse based on each fill's brightness, so labels are always readable. - SVG paint-order: stroke fill on labels with a 2px stroke in var(--bg-primary) for legibility on edge-case fills. - Theme-aware legend pill: replace hardcoded rgba(15,23,42,0.92) background with var(--bg-secondary) and var(--border) so it stops looking foreign in light mode. - Verified data-theme detection: confirm getGraphPalette() actually returns the right palette on theme toggle (current implementation may have a bug — the screenshot showed dark colors on a light background). Differentiation for features-vs-plans within the grayscale tier: dashed stroke on plan nodes. Reads as 'these are blueprints, not built yet' which matches the semantics. Built on top of an interim commit that ships the existing dark-mode-only color/theme/legend wiring as foundation. This feature replaces the palette portion entirely (the dark mode colors will become accent-tied, the broken light mode palette will go away), but keeps the data-graph-type legend wiring, getGraphPalette/paintGraphLegend helpers, theme MutationObserver, and CHARGE_STRENGTH=-220 from the foundation commit.
