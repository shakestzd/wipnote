Plan: Hook Noise Audit

Stop auto-creating spikes from hooks — use session events instead. Reserve spikes for deliberate investigations only.

In Progress 5 slices · ~10 files · Created 2026-03-30

Dependency Graph

A. Design DiscussionPending

Current State

Hooks automatically create spikes for errors, task results, and status changes. This has led to 644+ stale todo spikes that obscure real work items. The internal/hooks/ handlers call spike creation on various events, and all 10 agent definitions in plugin/agents/ included completion spike creation.

Desired End State

Spikes are only created deliberately via CLI (wipnote spike create) or orchestrator decisions. Hook events flow to the session event stream instead. The .wipnote/spikes/ directory contains only ~20 spikes with real architectural/investigation value.

Relevant Patterns

  • Session event stream — internal/hooks/session.go
  • Spike collection — internal/workitem/spikes.go
  • Hook handler pattern — internal/hooks/handler.go
